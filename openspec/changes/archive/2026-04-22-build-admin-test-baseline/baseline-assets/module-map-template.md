# 模块地图模板

## 1. 使用边界

- 首版全量测试基线构建：建立完整模块清单、关键链路、跨模块依赖和基础回归风险标签
- 后续迭代差异分析：仅在模块边界、入口、关键链路、依赖关系或关联规则发生结构变化时更新本模板
- 若本次变化只影响既有规则行为，不改变模块结构或依赖关系，则只更新 `iteration-diff-analysis-template.md` 与相关执行层模板

## 6. 更新规则

- 出现新模块、模块拆分合并、关键入口变化、跨模块依赖变化时，更新本模板
- 若仅原有规则逻辑调整但模块边界不变，可只更新规则库和差异分析
- 若结构变更来自版本需求，建议同步记录 `本次更新来源类型`、`来源 ID/回指` 和 `来源摘要`

## 7. 填写建议

- 一个模块至少要能说明“入口、对象、主流程、异常路径、依赖面”
- 风险标签建议从以下维度选择：`权限敏感`、`配置驱动`、`状态流转复杂`、`异步处理`、`跨系统`、`批量操作`
- 若模块层级较深，可继续扩展三级模块列，但保留统一的 `模块 ID` 体系
