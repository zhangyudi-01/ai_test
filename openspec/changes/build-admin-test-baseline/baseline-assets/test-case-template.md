# 测试用例模板

## 1. 使用边界

- 首版全量测试基线构建：本模板只用于定义字段和追溯要求，不直接生成真实业务测试内容
- 后续迭代差异分析：本模板必须显式承接 `RULE` 和/或 `DIFF`，并把验证结果传递给自动化映射和测试报告

## 9. 备注

- 每条用例至少应具备 `模块 ID`、`来源类型`、`来源 ID/回指`
- 后续迭代场景必须至少承接一个 `规则 ID` 或 `差异项 ID`
- 规则驱动场景必须填写 `规则 ID`
- 已进入自动化评估或自动化执行的场景，建议填写 `关联自动化项 ID`
