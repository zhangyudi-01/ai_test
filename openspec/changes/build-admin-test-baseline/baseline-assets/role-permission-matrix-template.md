# 角色权限矩阵模板

## 1. 使用边界

- 首版全量测试基线构建：建立角色、权限项、数据范围和流程限制的完整基线
- 后续迭代差异分析：仅在角色定义、权限项、数据范围、菜单入口、流程动作发生变化时更新本模板
- 若本次版本只复用既有权限模型做回归，不改变权限结构，则只更新 `iteration-diff-analysis-template.md` 与下游执行层模板

## 6. 更新规则

- 出现新角色、新权限项、数据范围变化、字段级限制变化、流程动作变化时，更新本模板
- 若仅某个版本验证既有权限路径，不修改权限结构，则只更新差异分析和执行层模板
- 若权限变化与规则变化同时出现，建议同步填写 `关联规则 ID` 与 `关联差异项 ID`

## 7. 填写建议

- 权限不是只有按钮维度，必须同时考虑菜单、接口、数据范围和流程动作
- 有条件权限时，条件写清楚触发前提，不要只填“部分可用”
- `权限项 ID` 应能回指到差异项、规则或用例，不应只停留在备注中
