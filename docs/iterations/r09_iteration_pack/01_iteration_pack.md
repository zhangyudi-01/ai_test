## 1. 本轮识别规则

### 1.1 范围识别规则
- 主 PRD 中标题、页面名、模块名后带“（九）”的内容，默认属于本轮迭代范围。
- 本次只关注用户指定范围内的“绿色背景标注”和“删除线 + 绿色背景标注”内容。
- 如果绿色背景信息在 AI 读取环境中不可稳定识别，则以本 iteration pack 中显式标签为准。
- 未带“（九）”且未在本轮模块内被引用的内容，默认作为历史基线或上下文，不作为本轮直接变更。

### 1.2 标签定义
- `[本轮新增]`：本轮新增页面、字段、规则、按钮、状态、动作、数据联动或流程。
- `[本轮修改]`：旧规则发生变化；即使只是在原句基础上加几个字，也归为修改。
- `[本轮删除]`：旧规则被删除、废弃或不再适用。
- `[沿用上下文]`：为理解本轮变更必须保留的历史规则背景。
- `[待确认]`：当前材料不足、外链/原型不稳定、或需要产品/开发确认的内容。

### 1.3 差异识别要求
后续生成 `02_diff_matrix.md` 时，必须基于本文件和主 PRD 输出：
- `DIFF ID`
- 所属模块
- 变更类型：新增 / 修改 / 删除 / 沿用
- 原行为 / 旧规则摘要
- 新行为 / 新规则摘要
- 影响旧路径
- 旧路径回归要求：必回归 / 抽样回归 / 不需要 / 待确认
- 风险等级
- 来源回指
- 待确认点

### 4.2 影响判断
- 异常原因由单层选择扩展为一级/二级结构时，所有展示页需检查是否按 `一级原因-二级原因` 展示。
- 监控记录新增预警、异常确认中“异常原因”应复用该枚举与排序。
- 告警详情、预警详情、备注弹窗如展示异常原因，应检查一级/二级拼接与空值处理。

## 5. 本轮跨模块链路

| 链路 | 涉及模块 | 本轮风险 |
|---|---|---|
| 填写监控记录 -> 事件/告警/预警 | 比赛监控、事件管理、告警管理、预警管理 | 新增异常生成比赛异常事件与告警；新增预警生成事件与预警记录 |
| 告警核实 -> 异常确认 -> 变更管理 | 告警管理、变更管理 | 告警确认为比赛异常后推送异常变更，并可能更新标准化比赛信息 |
| 赛果审核 -> 赛果确认 -> 批量确认 -> 赛果回退 | 比赛监控、赛果确认记录 | 赛果状态、比分、主客队顺序、确认人员/审核人员均受影响 |
| 异常变更处理 -> 赛果确认记录 | 变更管理、比赛监控 | 异常变更可能成为赛果确认记录来源，并影响比分/比赛状态 |
| 基础信息外链/球队关联 -> 比赛列表/草稿/告警/预警展示 | 基础信息管理、比赛监控、异常管理 | 赛事/球队名称、官网媒体链接、北单全称/简称显示联动 |

## 6. 本轮高风险点

- 主客队顺序和比分顺序在比赛列表、赛果审核、赛果确认、批量确认、赛果回退、告警核实、异常确认、预警详情等多个页面之间联动。
- “新增异常/新增预警”不是单点保存，而会联动生成事件、告警、预警。
- 告警确认后可能更新标准化比赛状态，并推送变更管理。
- 异常变更中退票、按实际结果开奖、开奖延后、调整停售时间等操作会影响赛果、开奖时间、变更状态和导出文件。
- 关联球队中北单全称/简称可能回写数据中心球队信息，影响赛事/球队展示。
