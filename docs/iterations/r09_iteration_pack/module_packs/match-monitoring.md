## 2. 比赛列表（九）

### 2.1 查询条件

[本轮新增]
- 奖期：下拉选择，包含系统所有奖期编号及开售游戏，按奖期 ID 倒序排列；默认选中当前用户之前选择的奖期，未选择时默认选中 ID 最大奖期。
- 自动刷新：下拉选择，包含 `关、5s、10s、30s、1m、5m`，默认 `关`。
- 运动项目：支持多选，默认全部。
- 编号范围：仅数字，最多 4 位，默认空，支持相同编号。
- 开赛时间：开始/结束时间，显示到分；选择日期后默认 `00:00:00-23:59:59`。
- 游戏：`全部、足球单场、胜负过关、下半场比分`，从赛程开售游戏中查找，默认全部。

### 2.2 列表结果与字段展示

[本轮修改]
- 查询结果为对应奖期下所有已开售批次的正式版比赛列表。
- 排序：按置顶状态、比赛编号正序。
- 不分页。

[本轮修改]
- 编号：取比赛编号流水号，隐藏前几位 0，例如 `25051-0001-A123456` 显示为 `1`。

[本轮修改]
- 赛事简称：取赛程信息中比赛所属赛季对应的北单简称；为超链接样式，具体规则同赛程草稿。

[本轮修改]
- 主队全称、客队全称：
  - 取标准化比赛信息中的球队；
  - 名称优先显示比赛所属赛季关联的球队北单全称；
  - 为空时显示球队全称；
  - 为超链接样式，具体规则同赛程草稿。

[本轮新增]
- 主客队顺序根据赛果状态变化：
  - 赛果未审核：主客队顺序与标准化比赛信息一致；
  - 赛果已审核：主客队顺序与当前用户审核结果一致；
  - 赛果已确认：主客队顺序与确认结果一致。

[本轮修改]
- 主队排名、客队排名：取标准比赛信息排名，无则不显示；排名顺序与主客队一致。

[本轮新增]
- 足球项目比赛中状态显示贝泰时间：
  - `match_clock` 为空：显示 `match_status`；
  - `match_clock` 非空且 `stoppage_clock` 为空：显示 `match_clock` 分钟，如 `10'`；
  - 二者均非空：显示 `{match_clock}+{stoppage_clock}`，如 `45'+1'`。

[本轮修改]
- 半场/全场比分展示优先级：已确认比分 > 当前用户已审核比分 > 系统采集开奖用比分 > 无比分显示 `-`。
- 比分顺序与主客队一致。

[本轮新增]
- 赛果状态：未确认时显示当前用户审核状态；已确认时所有用户均显示为已确认。
- 图：已上传赛果截图高亮，否则置灰。
- 告警：显示对应比赛未确认状态告警条数；0 时置灰。
- 变更：显示对应比赛待执行和执行中状态变更条数；0 时置灰；不含系统自动处理任务。
- 预警：显示对应比赛预警条数；0 时置灰。
- 事件：显示对应比赛类型为比赛异常、预警，异常渠道为贝泰/纳米/同行，且时间节点为开售后的事件条数；0 时置灰。
- 备注：显示对应比赛备注条数。
- 监控记录：显示当前用户填写的该比赛监控记录最后更新时间，无记录显示 `-`。
- 置顶：显示当前用户置顶状态。

### 2.3 行背景色规则

[本轮新增]
- 告警列数值大于 0 时，行背景填充为对应告警记录中最高告警级别颜色。

[本轮新增]
- 比赛存在以下事件时，行背景填充对应颜色：
  - 开售后 + 比赛提前：红色；
  - 开售后 + 比赛推迟：黄色；
  - 开售后 + 比赛延期/取消/中断/腰斩：橙色；
  - 人工添加事件与数据源事件同时存在时，优先按事件更新时间最大的人工添加事件填充，否则按事件更新时间最大的数据源事件填充；
  - 无上述事件但列表中主客队顺序与赛程不一致时，填充绿色；一致后取消填充。

[待确认]
- 告警背景色与事件背景色同时满足时的优先级。

### 2.4 自动刷新与筛选

[本轮新增]
- 比赛列表根据自动刷新时长自动更新。
- 自动刷新只更新列表内数据，保留查询/筛选结果和滚动条位置。

[本轮新增]
- 列表标题筛选项包括：赛事简称、比赛状态、赛果状态、告警、变更、预警、事件、备注、开奖状态。
- 各筛选项默认全选，部分支持反选。

### 2.5 列表交互

[本轮新增]
- 点击赛事简称、主队全称、客队全称外链，在新标签页打开对应网站，同时记录 1 条当前用户对该比赛的监控记录，记录内容为“无变动”。
- 点击比赛状态，弹出对应比赛进程信息。
- 点击图，左侧显示对应比赛截图。
- 点击告警，打开告警管理页并显示对应比赛未确认状态告警记录。
- 点击变更，打开变更管理页并显示对应比赛待执行和执行中状态变更记录。
- 点击预警，打开预警管理页并显示对应比赛预警记录。
- 点击事件，打开事件管理页并显示对应比赛事件记录。
- 点击备注，弹出备注框。
- 点击置顶图标，切换当前用户置顶状态；切换后刷新当前列表排序，页面定位不变。

## 3. 填写监控记录（九）

[本轮新增]
- 用户勾选比赛后点击【填写监控记录】，弹窗显示记录填写框，包含所选比赛列表。
- 所选比赛中包含相同比赛 ID 时，仅显示一条，编号显示其中最小编号。
- 按编号正序排列。
- 信息显示规则与监控列表一致。

[本轮新增]
- 监控记录类型：无变动、新增异常、新增预警，默认无变动。

### 3.1 新增异常

[本轮新增]
选择新增异常时展开其他输入项，均必填：
- 异常类型：请选择、比赛提前、比赛推迟、比赛延期、比赛中断、比赛取消、球队变更、赛果争议；
- 异常渠道：请选择、联赛官网、当地媒体、俱乐部官网、官方社交媒体；
- 异常详情：支持汉字、字母、数字和字符，长度 ≤ 300；
- 告警级别：请选择、红色、橙色、黄色；
- 变动后开赛时间：比赛提前时显示，支持编辑到分；比赛推迟/比赛延期时显示，可选择待定或输入具体时间，具体时间必填，支持编辑到分；
- 变动后比分：赛果争议时显示，足球项目输入半场和全场比分，非足球项目输入全场比分；比分范围 `[0,999]`，全场比分需大于等于半场比分，比分顺序与主客队一致。

### 3.2 新增预警

[本轮新增]
选择新增预警时展开：
- 异常原因：级联下拉，具体内容及排序见通用需求异常原因列表；
- 异常渠道：请选择、联赛官网、当地媒体、俱乐部官网、官方社交媒体；
- 异常详情：支持汉字、字母、数字和字符，长度 ≤ 300。

### 3.3 提交联动

[本轮新增]
- 点击【提交】，保存当前用户对应比赛的监控记录。
- 新增异常：添加类型为比赛异常的事件，并生成告警记录，当前用户核实状态置为已核实。
- 新增预警：添加对应事件，并生成预警记录。
- 点击【取消】关闭弹窗，不更新数据。
- 返回比赛列表时保留查询、筛选和滚动条位置。

[待确认]
- “当前用户的核实状态置为已核实”作用对象是告警记录还是比赛记录。
- 事件内容参考《数据采集系统 PRD 文档》，需补充事件字段映射。

## 4. 赛果审核（九）

[本轮新增]
- 勾选比赛点击【赛果审核】，弹窗显示审核界面，默认展示所选比赛中编号最小比赛详情。
- 比分显示系统采集的开奖用比分，比分顺序与主客队一致。
- 编号显示当前比赛 ID 在当前奖期所有已开售正式版赛程中的编号。
- 其他内容显示规则同比赛监控列表页。

[本轮新增]
- 若当前用户已审核过赛果，无论是否已确认，回显审核时填入的核实内容和主客队切换图标状态；否则为空。
- 点击主客队切换图标时互换主客队位置，比分顺序同步；互换后按钮高亮；再次点击恢复。

[本轮新增]
- 核实结果：赛果正确、赛果错误，默认赛果正确。
- 比赛状态非完场时，不可选择赛果正确。
- 选择赛果错误时可选择比赛状态：完场、推迟、中断、取消，默认完场。
- 选择完场时可编辑比分；足球输入半场/全场比分，非足球输入全场比分；范围 `[0,999]`；全场比分需大于等于半场比分。
- 输入的比赛状态及比分与比赛详情完全一致时不可提交。
- 选择其他比赛状态时不可编辑比分，足球半场/全场、非足球全场比分固定为 `-1:-1`。
- 核实渠道：联赛官网、当地媒体、俱乐部官网、官方社交媒体，默认联赛官网。

[本轮新增]
- 上一场/下一场：不保存当前修改，刷新为相邻编号比赛；边界比赛不可点击。
- 提交并继续：
  - 校验核实结果的主客队顺序与赛程是否一致；不一致提示“当前比赛的主客队顺序与赛程不一致，请修改后再提交”；
  - 校验通过后，更新当前用户对应比赛的赛果、主客队顺序，并更新赛果状态为已审核；
  - 刷新为下一场，若当前为奖期最后一场则不刷新。
- 赛果状态已确认或开奖状态已开奖时，不可编辑和提交核实内容。

## 5. 赛果确认（九）

[本轮新增]
- 勾选比赛点击【赛果确认】，弹窗展示所选比赛中编号最小比赛的赛果确认结果。
- 编号、运动项目、赛事简称、轮次、开赛时间、主客队全称显示同赛果审核页。

[本轮新增]
- 审核人数不足两人时（含无人审核）：
  - 提示“赛果审核的人数不足两人，请先进行赛果审核！”；
  - 主客队顺序、比赛状态、比分显示当前用户审核赛果，无则显示系统采集开奖用比分；
  - 【确认并继续】置灰。

[本轮新增]
- 审核结果不一致时：
  - 列出所有审核人员及各自审核赛果；
  - 主客队顺序与各自审核结果一致；
  - 突出显示不一致信息；
  - 按审核时间倒序、审核记录 ID 倒序；
  - 【确认并继续】置灰。

[本轮新增]
- 审核结果一致时，合并显示审核人员及审核赛果。
- 当前赛果已确认或比赛已开奖时，提示“赛果已确认！”或“比赛已开奖！”，确认按钮置灰。
- 点击确认并继续：更新北单赛程比赛状态和比分，更新赛果状态为已确认，记录确认人员；确认结果为完场时同步更新标准化比赛状态和比分。

## 6. 批量确认（九）

[本轮新增]
- 勾选比赛点击【批量确认】，跳转赛果确认结果页。
- 先校验是否包含赛果状态为已确认的比赛：包含则提示“所选比赛中包含“已确认”状态的比赛，请移除后再试”，终止确认。
- 校验各比赛审核人数：不足两人时提示“编号为{编号}、...、{编号}的比赛，赛果审核的人数不足两人，请先进行赛果审核”，终止确认。
- 展示比赛数量、比对通过数量、比对未通过数量和比对明细。
- 比对明细区分通过/未通过标签；存在未通过时默认显示未通过，否则默认显示通过。
- 通过列表按编号正序，同一比赛信息合并展示。
- 未通过列表按编号正序，列出全部审核人员及审核赛果，突出不一致信息，按审核时间倒序、审核记录 ID 倒序。
- 比对未通过数量大于 0 时，【确认赛果】不可用。
- 点击查看可查看赛果截图，无截图时不可点击。
- 确认赛果后更新北单赛程状态和比分、赛果状态为已确认并记录确认人员；完场结果同步标准化比赛状态和比分。

## 7. 赛果回退（九）

[本轮新增]
- 勾选比赛点击【赛果回退】。
- 弹窗展示所选比赛信息及已确认赛果。
- 未确认赛果的比赛，比赛状态、比分均显示“未确认”。
- 点击提交：清空北单赛程比分、比赛赛果审核记录，更新赛果状态为未确认，并记录回退人员。
- 点击返回关闭弹窗，不更新数据。
- 返回列表时保留查询、筛选和滚动条位置。

## 8. 赛果确认记录（九）

[本轮新增]
- 查询条件：奖期下拉，包含所有奖期编号及开售游戏，按奖期 ID 倒序，默认 ID 最大。
- 查询结果：对应奖期下所有已确认赛果的赛程列表，按比赛编号正序，不分页。

[本轮新增]
- 展示字段：编号、运动项目、赛事简称、轮次、开赛时间、主客队全称、比赛状态、半场/全场比分、审核人员、确认人员、来源。
- 主客队顺序与北单赛程赛果确认结果一致。
- 比赛状态：赛果确认流程确认的赛果显示用户审核录入状态；异常变更流程确认的赛果显示数据中心比赛状态。
- 审核人员/确认人员：根据来源显示赛果确认流程的审核/确认人员，或异常变更流程的告警核实人员/异常变更操作人员。
- 来源：赛果确认、异常变更。
- 点击查看显示赛果截图，无截图不可点击。

[待确认]
- 来源为“异常变更”的判定规则中提到退票/按实际结果开奖，需和变更管理最终处理状态口径确认。
