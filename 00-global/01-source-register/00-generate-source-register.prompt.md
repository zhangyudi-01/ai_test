# 提示词：生成上下文源注册表

## 任务说明

根据 `00-source-materials` 目录的内容，生成或更新上下文源注册表。

## 输入资料

**源材料目录**：`00-global/00-source-materials/`

扫描该目录下的所有子目录和文件，包括：
- 01-current-requirement（当前需求）
- 02-history-requirements（历史需求）
- 03-history-scheme（历史方案）
- 03-history-testcases（历史用例）
- 05-api-docs（API文档）
- 06-database-docs（数据库文档）
- 07-history-defects（历史缺陷）
- 08-business-modules（业务模块）
- 09-human-understanding（人工理解）

## 输出要求

生成文件：`context-source-register.md`

存放位置：`00-global/01-source-register/`

## 注册表格式要求

### 文档结构

```markdown
# Context Source Register

## 说明
本注册表记录所有可用的上下文资料源，用于AI分析和测试设计时的资料索引。

## 资料分类说明
- **权威级别**：高（最终依据）、中（参考资料）、低（背景信息）
- **使用范围**：明确该资料在何种场景下使用
- **禁止用途**：明确该资料不能用于何种目的

---

## 注册表

| Source ID | 资料类型 | 文件路径 | 资料名称 | 版本/日期 | 权威级别 | 使用范围 | 禁止用途 | 备注 |
|---|---|---|---|---|---|---|---|---|
| ... | ... | ... | ... | ... | ... | ... | ... | ... |

---

## 资料优先级说明
...

## 阶段使用策略
...
```

### 字段说明

1. **Source ID**：格式为 `SRC-XXX`（三位数字，如 SRC-001）
2. **资料类型**：当前需求、历史需求、历史方案、历史用例、API文档、数据库文档、历史缺陷、业务模块文档、人工理解
3. **文件路径**：相对于项目根目录的路径（如：`00-global/00-source-materials/01-current-requirement/xxx.md`）
4. **资料名称**：简短的资料描述
5. **版本/日期**：资料版本号或日期，如无则标注"-"或"待补充"
6. **权威级别**：
   - **高**：最终依据，不可被其他资料覆盖
   - **中**：参考资料，用于补充理解
   - **低**：背景信息，仅供参考
7. **使用范围**：明确说明该资料在何种场景下使用
8. **禁止用途**：明确说明该资料不能用于何种目的
9. **备注**：其他重要说明

### 权威级别判定规则

- **高**：当前需求、API文档、数据库文档
- **中**：历史需求、历史方案、历史用例、历史缺陷、业务模块文档、人工理解
- **低**：附件、辅助资料、SQL查询等

## 特殊处理要求

### 1. 待补充资料

如果目录存在但文件缺失（如 `05-api-docs/` 为空），按以下格式记录：

```
| **SRC-013** | API文档 | 00-global/00-source-materials/05-api-docs/ | API接口文档 | 待补充 | **高** | 接口测试设计依据 | - | **待开发补充** |
```

### 2. 业务模块文档

如果 `08-business-modules/` 待生成，标注：

```
| **SRC-017** | 业务模块文档 | 00-global/00-source-materials/08-business-modules/ | 业务模块说明文档 | 待生成 | 中 | 理解业务模块划分 | 不作为需求依据 | **待AI生成** |
```

### 3. 历史资料附件

对于 `attachments` 子目录中的文件，单独列为资料条目。

## 阶段使用策略

在注册表末尾添加各阶段的资料使用策略：

### 1. 项目全局上下文阶段
- 主要使用：SRC-001（当前需求）、SRC-017（业务模块）、SRC-018（人工理解）
- 历史资料只作背景索引

### 2. 原子需求拆分阶段
- 主要使用：SRC-001（当前需求）
- 历史资料用于辅助识别回归影响

### 3. 模块逻辑拆解阶段
- 主要使用：SRC-001 + SRC-013/014（API和DB）+ 历史方案
- 资料冲突时标记待确认

### 4. 测试方案和测试用例阶段
- 主要使用：确认后的需求基线
- 历史用例仅参考格式
