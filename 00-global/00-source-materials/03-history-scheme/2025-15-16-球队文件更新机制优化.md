### 功能测试

| 序号 | 测试项 | 测试点 | 备注 |
| --- | --- | --- | --- |
| 1 | matchup 文件 （改动） | **字段验证-LEAGUE\_ID：** 检查 `LEAGUE_ID` 是否正确关联了 `schedule` 表中的 `league_id`，且不为空 | 关联表：`schedule` 字段：`league_id` |
| 2 |  | **数据范围验证：** 验证文件内的比赛是否严格属于（统计日期-3个月）内的奖期，且无多余数据 | 依赖 `game_draw.draw_time` 过滤 |
| 3 | team 文件 （逻辑变更） | **字段验证-LEAGUE\_ID：** 检查 CSV 中所有记录的 `LEAGUE_ID` 是否固定为空（NULL/空字符串） | **PRD变更点**： 强制置空 |
| 4 | league 文件 （逻辑变更） | **关联准确性：** 验证文件中的 `LEAGUE_ID` 是否仅包含 `matchup` 文件中出现的赛事 ID | 去重逻辑验证 |
| 5 | game\_draw\_stats\_prz\_lvl 文件 （新增文件） | **文件生成验证：** 确认在 ZIP 包中生成了名为 `game_draw_stats_prz_lvl+日期.csv` 的文件 | 新增文件 |
| 6 |  | **数据精度验证：** 重点验证 `TOTAL_CHANCES`、`WIN_SP` 等字段的小数位精度，是否与 DB 中 `decimal` 类型一致，无精度丢失 | 涉及金额和注数 |
| 7 |  | **数据量验证：** 验证该文件的记录数是否与符合条件的 `game_draw` 数据量匹配 |  |
| 8 | total\_record 文件 （统计更新） | **汇总项验证：** 检查文件中是否新增了 `game_draw_stats_prz_lvl` 的条目 | 必须包含新文件 |
| 9 |  | **计数准确性：** 核对 `game_draw_stats_prz_lvl` 后面的数字是否等于实际生成的 CSV 行数（不含表头） | 验证后端计数器逻辑 |
| 10 | 整体回归 （轻量级） | **时间范围一致性：** 抽取奖期赛程等 CSV 文件，确保它们包含的数据都是基于同一个“统计日期”计算的（即数据快照一致） | 避免跨天执行导致的数据偏移 |
| 11 |  | **打包与清理：** 验证 ZIP 包命名是否正确（`BJ_日期.zip`），且旧文件清理逻辑（N+1天）依然生效 | 保证原有运维逻辑未被破坏 |
| 12 | 验证手动推送 |  |  |
| 13 | 关注下zip文件大小，与之前对比 |  |  |
