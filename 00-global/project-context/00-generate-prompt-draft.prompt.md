# 提示词：生成项目全局上下文提示词草稿

你现在要生成一份用于"项目全局上下文理解"的专业提示词草稿。

## 任务说明

**注意**：本阶段不是详细需求分析，不做原子需求拆分，不生成测试方案，不生成测试用例。

## 阶段目标

本阶段目标只是让 AI 基于当前资料，形成一份粗粒度项目背景理解，帮助后续流程知道：

1. 这是一个什么项目；
2. 当前需求大致属于什么业务范围；
3. 有哪些可用资料；
4. 哪些资料是权威依据；
5. 哪些资料只能参考；
6. 哪些资料待补充；
7. 后续详细分析应该在哪些阶段进行。

## 输入资料

1. `context-source-register.md`
2. `raw-requirement.md`
3. `REQ-当前迭代需求-v1.md`
4. `human-understanding.md`
5. 如有业务模块说明，可参考业务模块说明

## 输出要求

生成文件：`project-context-prompt.draft.md`

存放位置：`00-global/project-context/`

## 提示词要求

生成的提示词必须满足以下要求：

1. ✅ 明确禁止详细拆分需求
2. ✅ 明确禁止生成测试点和测试用例
3. ✅ 明确禁止用历史资料覆盖当前需求
4. ✅ 明确当前需求是最高依据
5. ✅ 明确项目上下文只做粗粒度理解
6. ✅ 明确输出内容应控制在简洁范围
7. ✅ 明确不确定内容必须标记为待确认
8. ✅ 明确历史资料只作为背景索引，不展开详细分析
